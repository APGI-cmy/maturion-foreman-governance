A formal FAIL-ONLY-ONCE rule should be added: "Submitting agent MUST verify branch push state via GitHub API (not just local git) before invoking IAA. This eliminates the pattern of agents invoking IAA with a false "all committed" claim.

3. **Correction addendum lifecycle management**: Correction addenda are accumulating (R2 addendum is stale at R4 invocation). A policy should require that each re-invocation round produces a FRESH correction addendum that (a) supersedes all prior addenda and (b) is accurate to the CURRENT git state at time of invocation — not the expected future state after a pending push.
