## Lessons

### What Worked Well
- Fetching `origin/main` and computing `git diff --name-status` against the branch revealed the true scope of the PR (35+ files vs. stated 7). This is the correct approach for any canon-touching PR.
- Prior session memories on main (readable via `git show FETCH_HEAD:`) provided crucial context about PR #1261 and the IAA contract upgrade history.

### What Was Challenging
- The shallow clone (grafted history) made the first-commit `git show` uninformative. The correct approach was fetching main and diffing against it.
- The branch-behind-main pattern is subtle: the PR description looks legitimate and even CS2 agreed with the content ("Technical Changes Verdict: ✅ All AC Met"), but the git diff revealed catastrophic governance regressions.

### What Future Sessions Should Know
- ALWAYS fetch origin/main and compute `git diff FETCH_HEAD..HEAD --name-status` before any assessment. Never rely solely on the PR description's stated file list.
- When a PR includes agent contract files in the diff (even if not in the stated scope), HALT-004 / Rule 4 applies immediately.
- Branch-behind-main contamination is a high-risk pattern that can disguise simple, correct content changes behind dangerous governance regressions.

### Governance Insights
- The IAA contract v2.0.0 upgrade (PR #1261) was correctly rejected by IAA on independence grounds and then handled by CS2 directly. The current main has v2.0.0 as the result. This session validated that the IAA contract governance model (CS2-only for IAA contract changes) is functioning correctly.
