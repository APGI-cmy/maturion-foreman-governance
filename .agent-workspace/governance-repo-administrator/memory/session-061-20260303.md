## Learning Loop

**Recurring shortfall (2nd instance)**: Rule A-09 violated again (PR #1255 was first instance).
This is now a pattern — GA repeatedly submits report_progress without IAA invocation first.

**Root cause analysis**: The `report_progress` workflow is natural for incremental progress
reporting, but it creates PRs/commits without a natural "gate" step before the first commit.
The A-09 obligation must be applied BEFORE the FIRST report_progress call that creates a PR.

**Improvement suggestion** (parked): Add a pre-flight checklist item at the START of every GA
session: "Has the evidence bundle been created? Has IAA been invoked? If yes to evidence +
invocation → proceed with report_progress. If no → create evidence bundle first, THEN invoke
IAA, THEN use report_progress." This makes the IAA invocation a pre-condition for report_progress,
not a post-condition.

This is the 2nd instance of A-09 violation. Per FAIL-ONLY-ONCE invariants (INV-603, OVF-002),
this pattern should be considered for promotion to a new Conditional Rule in the next session.
