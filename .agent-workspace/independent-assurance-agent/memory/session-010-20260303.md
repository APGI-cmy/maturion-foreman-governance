### Improvement Suggestion 1 — R2 Invocation Protocol: CS2 Draft-State Waiver Pattern

**Observation**: The R2 invocation included an explicit CS2 instruction to "record the isDraft state and issue verdict accordingly." This created a novel handling pattern: IAA proceeds with full assessment despite BLOCKING INV-409, and issues ASSURANCE-TOKEN with a CS2 merge-prerequisite notation.

**Concrete suggestion**: This pattern should be codified in the session-memory-template or IAA trigger guidance as a named exception: "CS2 Draft-State Waiver." When CS2 explicitly authorizes assurance to proceed despite draft state, the draft state is noted as a merge prerequisite (not blocking), and the verdict reflects completed assurance with CS2 action required before merge. This prevents future IAA sessions from issuing an unnecessary REJECTION-PACKAGE when CS2 has already acknowledged the draft state as a platform constraint.

**Where to codify**: Add a note to iaa-category-overlays.md or FAIL-ONLY-ONCE.md: "INV-409 exception: If CS2 explicitly authorizes R2+ invocation while PR is in DRAFT, record draft state as merge prerequisite and proceed to ASSURANCE-TOKEN if all other checks pass."

**Priority**: HIGH — this pattern will likely recur for any CodexAdvisor/copilot-swe-agent PR until the tooling constraint is resolved.
