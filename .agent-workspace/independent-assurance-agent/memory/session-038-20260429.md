## Learning Loop

### Observation 1 — Tier 1 vs Tier 2 Trigger Table Tension (Recurring — sessions 036, 037, 038)

**Pattern**: Three consecutive sessions have noted the tension between the IAA canon (Tier 1)
classifying T3/T4 PRs as "IAA NOT required" and the Tier 2 trigger table classifying
CANON_GOVERNANCE and CI_WORKFLOW as "MANDATORY." This session resolved it via A-003 (ambiguity →
mandatory invocation) + CS2 explicit invocation.

**Improvement suggestion**: The Tier 2 trigger table should be updated to align with the Tier 1
Risk-Tiered Ceremony Table. CANON_GOVERNANCE trigger should note "T3 — IAA NOT required by default
per canon; CS2 Direct Review sufficient; IAA invoked when CS2 explicitly requests or when combined
with T1/T2 triggers." CI_WORKFLOW should similarly note "T4 — IAA NOT required by default." This
would prevent future sessions from facing this ambiguity.

**Promotion flag**: Third consecutive session encountering this pattern → ESCALATION NOTE created
for CS2 per OVF-003. See escalation-inbox entry.

### Observation 2 — CANON_INVENTORY Update Discipline for Copilot Coding Agent PRs

**Pattern**: The Copilot coding agent PR modified three canon files but did not update
CANON_INVENTORY.json. This is a recurring gap — the Copilot coding agent does not have
built-in awareness of the CANON_INVENTORY maintenance requirement.

**Improvement suggestion**: The SCOPE_DECLARATION_SCHEMA.md (being updated in this very PR) should
include an explicit requirement that any Copilot-agent PR modifying governance/canon/* files must
also update CANON_INVENTORY.json. Additionally, the CI gate (governance-scope-to-diff-gate.yml or
a new gate) could validate that any PR modifying governance/canon/* has a corresponding
CANON_INVENTORY update. This would prevent the pattern from recurring for future Copilot-agent PRs.

### Observation 3 — T3 PREHANDOVER Proof Requirement

**Pattern**: For T3 Copilot-agent PRs explicitly invoked by CS2, a PREHANDOVER proof is technically
required by CORE-018 but the Copilot coding agent doesn't natively produce one. This creates
overhead for legitimate T3 PRs.

**Improvement suggestion**: The IAA canon and CORE-018 checklist should either (a) explicitly carve
out T3/T4 Copilot-agent PRs from the PREHANDOVER proof requirement when CS2 is directly reviewing,
or (b) define a minimal "T3 evidence artifact" template that the Copilot agent can create without a
full Foreman-style PREHANDOVER ceremony. Currently the Copilot agent must create a PREHANDOVER-like
artifact, which adds ceremony overhead to lightweight T3 PRs.
