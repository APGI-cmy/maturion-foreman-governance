### Concrete Improvement Suggestions

1. **Scope Declaration Authoring Guidance for Multi-Invocation PRs**: The producing
   agent pre-declared R3 artifacts (rejection-package-1315-r3.md, session-020-20260321.md)
   in the scope declaration. This is a forward-looking declaration that creates phantom
   entries when the anticipated invocation doesn't complete. Recommendation for
   FAIL-ONLY-ONCE or CodexAdvisor Tier 2 knowledge: explicitly prohibit pre-declaring
   future ceremony artifacts in scope-declaration.md. Scope declarations should only
   contain files already committed to the branch at the time of the declaration update.

2. **A-028 Clarity on Groupings**: The producing agent interpreted the R2 A-028 remediation
   as "change code block to list" — missing that the grouping headers ("Primary deliverables:",
   "Ceremony artifacts:") are also prohibited. The A-028 rule text says "no groupings" clearly,
   but in practice agents distinguish "list vs code block" and not "flat list vs grouped list."
   Recommendation: Add an explicit example to the Tier 2 knowledge for A-028 showing a WRONG
   (grouped list) vs RIGHT (flat list) format. This would prevent the partial-fix pattern seen
   in R2→R4.

3. **Post-Timeout Re-invocation Checklist**: When a prior IAA session times out without verdict
   (as R3 did), there is no established guidance for what the producing agent should do before
   re-invocation. A procedure note in FAIL-ONLY-ONCE or IAA Tier 2 knowledge would help:
   "If prior IAA invocation timed out, verify scope-declaration.md does not contain files
   pre-declared for the timed-out session that were never created."

4. **A-024 (`secret:` field) Systemic Observation**: `governance-repo-administrator-v2.agent.md`
   has `secret: MATURION_BOT_TOKEN` (pre-existing). IAA's own contract has `secret: "[REDACTED]"`.
   A-024 prohibits `secret:` in agent contracts. Both fields pre-date this PR and appear to have
   been accepted in prior assurance sessions without flags. Recommendation: CS2 should schedule
   a remediation PR to rename all `secret:` fields to `secret_env_var:` across agent contracts.
   This finding should be added to the IAA escalation inbox for CS2 review.
