triggers a REJECTION-PACKAGE for a CANON_GOVERNANCE class PR.

### Improvement Suggestion 2:
**Wave Checklist Gate protocol clarification needed for non-wave CANON_GOVERNANCE sessions**

The IAA Pre-Brief Protocol §IAA Invocation Gate says "mandatory for all qualifying PRs" but
the protocol trigger explicitly requires an active Foreman wave. This creates ambiguity for
standalone governance canonisation sessions (CANON_GOVERNANCE category, no wave active).

IAA-20260320-PR1300 is the first session to encounter this specific ambiguity. The current
NOT_APPLICABLE declaration was accepted as reasonable (vacuously satisfied gate), but the
protocol text does not explicitly provide a NOT_APPLICABLE carve-out.

**Concrete suggestion**: CS2 should add an explicit carve-out to IAA_PRE_BRIEF_PROTOCOL.md
§IAA Invocation Gate: "If no active wave exists (`.agent-admin/waves/` is empty), the gate
is N/A. Governance canonisation sessions outside a Foreman wave are exempt from the wave
checklist requirement."

This would eliminate ambiguity for future canonisation sessions.

### Improvement Suggestion 3:
**OVL-CG-004 (Ripple) should have an explicit NOT_APPLICABLE path in PREHANDOVER template**

governance-repo-administrator-v2 PREHANDOVER template has no dedicated ripple section for
CANON_GOVERNANCE deliveries. Adding an explicit ripple section to the template would prevent
